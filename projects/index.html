Machine Learning

Each Machine Learning project was made for an AI/ML class and completed in Python. The ML Models were created using the Tensorflow libraries, and data cleaning was done using Pandas and Numpy.
See the code for each ML project here!

Image Recognition

Trained on the Cifar-10 dataset, the goal was to train a Convolutional Neural Network that could correctly classify any one image from the dataset to one of ten textual descriptions.

N-Queens Solver *(using DEAP Python library)

The N Queens problem a.k.a Eight Queens Puzzle describes a challenge in placing N queens on an NxN chessboard such that no two queens can attack each other. The goal of this project was to build a Genetic Algorithm that will determine the placement of 8 neutral queens on an 8x8 chessboard.

Network Intrustion Detection

Trained on the 10% subset of the KDD ’99 dataset, the goal was to create a Fully Connected Neural Network that best predicts the nature of a network connection and is able to categorize each a connection within two descriptions: decidedly "legitimate" and otherwise “untrustworthy”.

Spam E-Mail Classifier

Trained on the Email Spam Classification dataset found on Kaggle, this project features the use of a Convolutional Neural Network and Recurrent Neural Network . This project inteded to address the fact that the digital footprint of the average person grows with the side effect of greater online presence. This ML Model was trained on our dataset to be able to classify SPAM email using tokenization (in the form of word segmentation) and TF-IDF

Yelp Business Ratings

Trained on the Yelp Academic dataset, the goal was to create a Natural Language Processor that could estimate the rating of a business out of 5 stars based on language used in the written reviews of that business. Language evaluation was done using TF-IDF statistics. These statistics derived from the words in business reviews essentially had the power to increase or reduce the star rating of a business if the language used was deemed to be "positive" or "negative" reinforcement reflecting customer experience.

Spider "Walk-It" Game

Made using the CodenameOne framework for Java as a semester long class project. As being the successor game-based project to Flappy Fish, this project was made with certain design patterns in mind: Singleton for the "Player", Iterator for game objects with the same types. Including abstract classes, inheritance, and interfaces (All sorts of Java specific terms but translates to other OOP languages). The game is availalbe for free. Peoplesense Dashboard

This project was completed as a capstone project with a group total of 8 members. The dashboard application is a web-based product which displays the current or historical occupancy counts of Amtrak trains in N. California. The dashboard fully customizable using widgets and users are tracked by Amtrak agencies via login authentication. This project was a complete overhaul of our client's existing solution and involved web development, database management, and project deployment for public use. Since the project was accepted by our client, the current owner of the codebase is Glocol Networks.
